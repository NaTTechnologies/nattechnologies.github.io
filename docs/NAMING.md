# Nombre oficial del proyecto

## Nombre adoptado

El nombre oficial es **Cafetal OS**.

En contextos técnicos se utilizan estas variantes:

| Uso | Valor |
|---|---|
| Nombre visible | `Cafetal OS` |
| Paquete npm | `cafetal-os` |
| Ejecutable | `cafetal-os` |
| App ID | `hn.cafetalos.community` |
| Base productiva | `cafetal-os.db` |
| Base demo | `cafetal-os-demo.db` |
| Carpeta documental de respaldos | `CafetalOS/Respaldos` |

## Justificación

El nombre reúne tres ideas:

- **Cafetal:** identifica de inmediato el sector y el lugar donde se originan los datos.
- **OS:** comunica una plataforma abierta y extensible, no únicamente una herramienta aislada.
- **Comunidad:** permite que productores, técnicos y desarrolladores participen en su evolución.

Cafetal OS no pretende presentarse como un sistema operativo general. En la comunicación pública debe describirse como una **plataforma open source para la gestión cafetalera**.

## Reglas de escritura

Correcto:

- Cafetal OS
- cafetal-os
- Comunidad Cafetal OS

Evitar:

- CafetalOS como nombre comercial visible;
- CAFETAL OS en textos corridos;
- Cafetal O.S.;
- AppCafé, AppCafe Honduras o nombres anteriores en nuevas interfaces.

## Migración desde versiones anteriores

Las claves antiguas de `localStorage` se leen únicamente para conservar preferencias del usuario durante la actualización. Los nombres técnicos antiguos no deben utilizarse para funciones nuevas.

Cada versión debe descomprimirse en una carpeta nueva para evitar que archivos compilados de una marca anterior permanezcan en `out/` o `node_modules/`.

## Validación de marca

La adopción del nombre dentro del repositorio no sustituye una búsqueda marcaria. Antes de una campaña comercial, registro formal o distribución internacional deben revisarse:

1. antecedentes de marca en los países objetivo;
2. dominio y organización de GitHub;
3. nombres similares en software agrícola;
4. reglas de uso del logo y forks;
5. titularidad y administración de la comunidad.
