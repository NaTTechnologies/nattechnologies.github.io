# Política de seguridad

## Versiones compatibles

Se prioriza la versión estable más reciente publicada en GitHub Releases.

## Reportar una vulnerabilidad

No publiques vulnerabilidades explotables en un issue abierto.

Utiliza GitHub Security Advisories cuando esté habilitado o escribe a:

```text
seguridad@nattechnologiesagency.com
```

Incluye:

- Versión afectada.
- Plataforma.
- Descripción del problema.
- Pasos mínimos para reproducirlo.
- Impacto observado.
- Evidencia sin datos reales.
- Propuesta de mitigación, cuando exista.

El equipo confirmará la recepción y evaluará severidad, alcance y corrección antes de divulgar detalles públicamente.
